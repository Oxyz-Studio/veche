Founder and software developer at O'XYZ Studio. We ship software, and for the past four months, vertical AI agents for the French justice system.

The problem

The most valuable software has no way in: no API, no DOM, no MCP. Citrix health records, mainframe green-screens, twenty-year-old desktop apps. Computer-use is the only door, but one agent is wrong about 1 in 5, and it forgets everything between runs.

OSWorld benchmark: the best computer-use models cap near 80%, so about 1 in 5 is wrong. On complex workflows it is far worse (UI-CUBE, UiPath).
Why me

In that world, reliability is not optional, it is the whole job: a wrong action has real consequences. Nothing out there was reliable enough, so I built it. VECHE earns trust by consensus, not by hoping a single agent is right.

How VECHE fixes it

Continual learning for computer-use.

A swarm of vision agents learns one shared map of the app by consensus, from pixels alone. A path is trusted only when independent agents agree, a small model then operates by reading the map, and the map heals itself when the software changes.

Reliable
consensus overrules a single agent's mistake, so the map beats any agent that built it.
Cheap
the small model just reads the map, no re-perceiving. Zero paid tokens on a known path.
Continual
it gets better the more it is used, and re-learns just the part that changes.
