How it works · no bullshit

A swarm learns to drive software it can only see.

No magic. It works from pixels, it trusts agreement over any single agent, it stores the result as a reusable map, it stays correct as the software changes, and it gets cheaper the more it's used. Here is each part, plainly.

01 · The input

It only ever sees a picture.

Every agent's single input is a screenshot. No DOM, no HTML, no accessibility tree, no API. To decide whether two screenshots are the same screen, it does not read any text on them: it turns each image into a numeric fingerprint (a multimodal embedding) and compares the fingerprints.

The same screen showing different data still matches; a genuinely different screen does not. That is why it works on a Citrix stream or a mainframe green-screen, where pixels are the only thing on the wire.

Measured: the same screen re-rendered at half size, larger, a squished aspect ratio, or a different window canvas all score 0.94 to 0.99 against the original, far above the match threshold. So it does not need a fixed resolution. The check itself is near-free, a hash or one embedding, not the expensive per-screen reasoning.
02 · Why a swarm, not one agent

One reading you can't verify is just a guess.

A single agent on a pixels-only app has nothing to check itself against, no API, no ground truth. So a wrong reading looks exactly like a right one. The only way out is redundancy: several independent agents look, and you trust what they agree on. Agreement is the only signal of correctness you can get without an oracle. Same principle as triple-redundant flight computers and error-correcting codes.

Each observation is a vote, weighted by how reliable that agent has proven. When one agent diverges, the majority overrules it and its trust drops, so it counts for less next time.

80%→99.7%

One agent on its own is right ~80% per step. Two that independently agree are right ~99.7%.

We measured this on the actual consensus code (Monte-Carlo). More agents mainly buy coverage, not accuracy. It assumes mistakes are independent; if a screen is confusing enough that everyone trips the same way, you raise the bar to three or more.
03 · The result

A reusable map, not a one-off run.

Every confirmed transition becomes an edge: from this screen, this action takes you to that screen. Put them together and you have a graph of the app, its screens and the actions between them.

That map is a portable artifact. Any computer-use tool can read it, whatever model it runs. It is, in effect, an MCP learned from pixels instead of authored by hand, and it can be served as a real one.

the MCP you never write
04 · Reuse

It gets cheaper the more it's used.

Once the map exists, you drop the expensive frontier model. A small cheap model just locates which screen it is on and follows the route, with no per-screen reasoning. The first traversal pays for the route; every reuse after is essentially free.

Cost per task falls roughly as the cold cost divided by the number of reuses, so a route that costs real money once becomes near-free at scale. And when the app changes, you only pay to re-learn the part that changed.

A screen the operator does not recognize triggers one targeted re-exploration, about the cost of a single cold run, paid once. After roughly the third reuse of that path, it has paid for itself.
05 · Staying correct over time

The map heals itself.

The map tracks its own confidence. A screen seen once is provisional and not trusted; it becomes confirmed only when a second independent agent agrees. So the map always knows what it is sure of, and what it is not.

When the software changes, the operator lands on a screen that matches nothing it knows, caught by the same fingerprint. It does not guess. It re-explores just that path, and consensus promotes the new screen once an independent agent confirms it; a transition that no longer holds loses votes as new evidence contradicts it. It re-learns the delta, not the app.

We show a provisional screen promoted to confirmed on real captured data (K=1 to K=2). The always-on online loop and the staleness decay run on an append-only log and are designed, not yet demonstrated at deployment scale. We would rather show the mechanism than fake the outcome.
06 · What's ours, what's borrowed

Honest about the novelty.

Fusing noisy votes into truth is well-established: Dawid-Skene truth inference, self-consistency, redundancy for reliability. We did not invent consensus, and we say so.

And here is the honest novelty. Pixels-only perception and per-agent memory are both commodity now (Agent S2 is vision-only with memory; AppAgent and Voyager accumulate reusable skills). What no one else does is the conjunction: a swarm that reaches consensus to merge one shared, model-agnostic map a small model reuses at amortized cost. Everyone else gives a single agent its own memory; VECHE makes many agents agree on one map anyone can use.
