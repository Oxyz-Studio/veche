The shift

Agents are learning to remember. The next step is learning together.

A single agent can now keep notes on an app and reuse them, but it learns alone, and it cannot tell its own mistakes from the truth. VECHE makes a swarm learn one app together, continually: every agent's experience flows into one shared map, agreement decides what is real, and the map keeps getting better the more anyone uses it.

The problem

Half the world runs on screens with no way in.

No API. No DOM. No MCP. Just pixels on a remote-streamed screen: Citrix-delivered health records, mainframe green-screens, twenty-year-old desktop apps. The software that runs hospitals, banks and governments was never built to be automated, and a coding agent cannot wrap it, because an MCP needs an API that simply does not exist here.

Why it's hard

One agent is a guess you can't check.

Today you do it by hand, or you trust a single AI agent. The best vision agents get a step right maybe eight times out of ten, and those errors compound over a long task. With no API to verify against, a wrong reading looks exactly like a right one. Nowhere near safe enough for money, health, or compliance.

The idea

Don't make one agent smarter. Make many agree.

A swarm of vision agents explores the app once and fuses what they see into one shared map by consensus. A path is trusted only when independent agents agree on it; when one diverges, it gets outvoted and its trust drops. The map ends up more reliable than any agent that built it. It is the same reason avionics runs three flight computers and votes.

01 · SEE

Pixels only

Each agent's only input is a screenshot. It recognizes a screen by its visual fingerprint, the same screen with different data is still the same screen.

02 · AGREE

Consensus map

Every observation is a vote, weighted by how reliable that agent has proven. Agreement filters out mistakes you could never otherwise catch.

03 · REUSE

Near-free at scale

Once the map exists, a tiny cheap model just follows it, no re-reasoning per screen. The first pass pays for it; every pass after is essentially free.

Continual learning, together

Memory isn't new. Agreeing on one shared map is.

Agents are starting to keep memory, but each keeps its own. VECHE's map is shared, and it stays correct by the same agreement that built it. A screen seen once is provisional until an independent agent confirms it. When the software changes, the operator lands where the map expected something else, so it does not guess: it re-explores just that path, and consensus promotes the new screen. It re-learns the delta, not the app, and never forgets the rest.

Honest scope: we show a provisional screen promoted to confirmed on real captured data. The always-on online loop and staleness decay are designed on top of an append-only log, not yet run at deployment scale. We would rather show the mechanism than fake the outcome.
